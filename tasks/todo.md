# 2026-05-10 首页操作与规则编辑收敛

## Spec

- 目标：把高频操作前移到首页列表，并把“全局规则”页收敛为只编辑两个现有全局文件的极简编辑器。
- 范围：首页列表操作区、按钮 loading / 禁用反馈、规则页 UI、规则扫描与保存 allowlist、相关测试和共享样式。
- 非目标：不新增新建规则、规则分类、项目级规则、拖拽排序、多文件批量编辑，也不改同步业务本身。
- 设计方向：首页保持紧凑工作区，但把“标记自制”和“删除”前移到列表行操作；规则页只保留 `~/.claude/CLAUDE.md` 与 `~/.codex/AGENTS.md` 的切换、编辑、预览和保存。
- 体验目标：
  - 首页用户无需打开详情即可完成同步、标记自制、删除等高频操作。
  - 所有耗时按钮都提供明确的 loading 文案和禁用态，避免重复点击和不确定感。
  - 规则页首屏不再出现新建、分类、override/rule 等无关概念，只专注编辑两个全局文件。
  - 缺失的全局文件会被明确标记为不可编辑，而不是继续提供创建入口。

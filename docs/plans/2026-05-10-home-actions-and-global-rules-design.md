# 首页操作与全局规则收敛设计

## 背景

当前首页把“标记自制”和“删除”放在详情抽屉中，导致用户需要先进入详情再做高频管理操作。规则页则仍保留新建、分类、Claude rules 和 Codex override 等概念，超过了当前产品需要。

用户本轮目标很明确：

- 耗时操作必须有 loading 反馈
- 标记自制和删除必须前移到首页列表
- 全局规则只支持 Claude 与 Codex 的主全局文件
- 整体 UI / 交互以简洁、顺手、少跳转为主

## 设计决策

### 1. 首页列表承担高频操作

首页保留现有紧凑表格，但将以下操作直接放到列表行操作区：

- 同步
- 标记自制 / 取消自制
- 删除
- 详情

这样详情面板退回为“补充信息查看”，而不是操作必经路径。

### 2. 所有异步按钮显式反馈

对同步、批量同步、标记自制、删除、保存规则等动作统一增加：

- pending 文案
- disabled 态
- 简单 spinner

目标不是增加动画感，而是让用户知道“点击已生效，正在处理”。

### 3. 规则页只编辑两个现有文件

规则页只保留：

- `~/.claude/CLAUDE.md`
- `~/.codex/AGENTS.md`

移除或隐藏以下概念：

- 新建规则
- Claude `rules/*`
- Codex `AGENTS.override.md`
- 类型筛选
- 分类和文件名配置

页面仅保留文件切换、编辑 / 预览 / 分栏、保存、草稿重置。

### 4. 后端允许范围同步收紧

为避免 UI 简化但后端仍保留旧能力，扫描和更新允许范围也一起收紧：

- 扫描模型只返回两个主文件
- 更新接口只允许改这两个文件
- 不再为新建规则提供产品能力

## 验收标准

- 首页可以不打开详情完成常见管理操作
- 所有异步按钮有明确 loading 反馈
- 规则页首屏不再暴露规则体系概念
- 扫描与更新测试覆盖新的允许范围
